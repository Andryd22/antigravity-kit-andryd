# /html-it — HTML output skill

**Stop reaching for markdown.** When the agent is asked to produce a doc, plan, report, review, design, prototype, dashboard, or editor — make it HTML instead. Higher information density, easier to read, easier to share, and (at levels 3-4) interactive.

> **Credit:** Framework and use-case taxonomy from **Thariq Shihipar** ([@trq212](https://x.com/trq212)) of the Anthropic Claude Code team. Original article: [Using Claude Code — The Unreasonable Effectiveness of HTML](https://x.com/trq212/status/2052809885763747935).

---

## When to reach for HTML over markdown

Markdown's default. HTML wins when **any** of these are true:

- Output is over ~100 lines (markdown stops getting read)
- Needs charts, tables, SVG diagrams, side-by-sides, or spatial layout
- Needs to be shared (HTML is a link; markdown is an attachment)
- Reader will manipulate it (sliders, toggles, drags, picks)
- It synthesises data from multiple sources
- User is writing university/academic notes

Markdown stays better for: tiny notes, commit messages, README first-pass, git diffs.

---

## The Four Levels

### Level 1 — Static Doc
HTML as better markdown. One-way, read-only.
**Use cases:** reports, explainers, plans, summaries.

### Level 2 — Visual Artifact
Adds SVG, tables, layouts. Still read-only.
**Use cases:** lecture notes, design systems, dashboards, comparison tables, slide decks.

### Level 3 — Two-Way Interactive
Sliders, toggles, drags. **Export button mandatory.**
**Use cases:** parameter tuning, A/B pickers, prompt iteration, flashcard review.

### Level 4 — Throwaway Tool
Purpose-built mini-app. Always ends with export back.
**Use cases:** triage boards, config editors, dataset curation tools.

### Typography rules
- H1: `font-family: var(--serif)`, `font-size: 40px`, `letter-spacing: -0.018em`, `line-height: 1.06`
- H1 key word: wrap in `<em>` → renders in italic clay colour
- Eyebrow above H1: mono, 11px, uppercase, `letter-spacing: 0.12em`, clay colour, preceded by 24px clay bar
- Body: 14.5px, `line-height: 1.68`
- Meta line: mono, 11.5px, var(--g500)
- H2: serif, 23px, border-bottom 1.5px var(--g300), margin-top 52px
- H3: sans, 13px, uppercase, letter-spacing 0.07em, var(--g700)

### Anti-patterns to avoid
- Linear gradients on everything
- Glassmorphism for documents
- External icon libraries (use inline SVG or skip)
- Loading spinners
- "Generated by AI on {date}" footers
- Animations longer than 200ms

### MathJax (LaTeX math in HTML)

Always include — user writes math in standard LaTeX syntax, MathJax renders it.
Config completa con macro da preamble.tex (già inclusa in `_preamble.html`):

```html
<script>
  window.MathJax = {
    tex: {
      inlineMath: [['\\(','\\)'], ['$','$']],
      displayMath: [['\\[','\\]']],
      tags: 'ams',          // numerazione automatica equazioni
      macros: {
        argmin: ['\\operatorname*{arg\\,min}'],
        argmax: ['\\operatorname*{arg\\,max}'],
        bm:    ['\\boldsymbol{#1}', 1],   // \bm{v} — vettore grassetto
        dv:    ['\\frac{d#1}{d#2}', 2],   // \dv{f}{x}
        pdv:   ['\\frac{\\partial #1}{\\partial #2}', 2], // \pdv{f}{x}
        R:     ['\\mathbb{R}'],
        N:     ['\\mathbb{N}'],
        Z:     ['\\mathbb{Z}'],
        E:     ['\\mathbb{E}'],
        P:     ['\\mathbb{P}'],
        norm:  ['\\left\\|#1\\right\\|', 1],
        abs:   ['\\left|#1\\right|', 1],
      }
    },
    svg: { fontCache: 'global' }
  };
</script>
<script src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-svg.js" async></script>
```

### Code block with syntax highlighting
Theme: Catppuccin Mocha (`background: #1E1E2E; color: #CDD6F4`).

```html
<div class="code-label">Python · NumPy</div>
<pre><span class="kw">def</span> <span class="fn">f</span>(x):
    <span class="cm"># commento</span>
    <span class="kw">return</span> x <span class="op">**</span> <span class="nm">2</span>
</pre>
```

Token classes:
- `.kw` → keyword (`def`, `return`, `import`) — `#CBA6F7` viola
- `.fn` → function name — `#89DCEB` cyan
- `.cm` → comment — `#6C7086` grigio italico
- `.st` → string — `#A6E3A1` verde
- `.nm` → number/literal — `#FAB387` arancio
- `.op` → operator — `#89DCEB` cyan

### SVG diagram
```html
<figure class="diagram">
  <svg viewBox="0 0 640 270" xmlns="http://www.w3.org/2000/svg">
    <rect width="640" height="270" fill="#FAFAF7" rx="8"/>
    <!-- elementi SVG: circle, rect, line, path, text -->
    <!-- frecce: marker-end="url(#arr)" + definisci marker in <defs> -->
    <defs>
      <marker id="arr" markerWidth="8" markerHeight="8" refX="6" refY="3" orient="auto">
        <path d="M0,0 L0,6 L8,3 z" fill="var(--slate)"/>
      </marker>
    </defs>
  </svg>
  <figcaption>Didascalia in mono 10.5px var(--g500)</figcaption>
</figure>
```

**SVG best practices:**
- Sempre `viewBox` + `width="100%"` — mai dimensioni fisse
- Background rect con `fill="#FAFAF7"` e `rx="8"`
- Grid lines: `stroke="#E8E5DC" stroke-width="0.8"`
- Usa i CSS custom properties nel fill/stroke quando possibile
- Label assi: `font-family="monospace" font-size="11" fill="#87867F"`

### Flashcards (Level 3 interactive)
```html
<div class="flashcards"> <!-- grid 2 colonne -->
  <div class="card" onclick="toggle(this)">
    <div class="card-q">Domanda?</div>
    <div class="card-a">Risposta. Formula \(\theta^*\) ok.</div>
    <div class="card-hint">↩ clicca per risposta</div>
  </div>
</div>

<script>
function toggle(card) {
  card.classList.toggle('revealed');
  card.querySelector('.card-hint').textContent =
    card.classList.contains('revealed') ? '↩ clicca per nascondere' : '↩ clicca per risposta';
}
</script>
```

### TOC active-section tracker (JS)
```js
const observer = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      document.querySelectorAll('.toc a').forEach(l => l.classList.remove('active'));
      const lnk = document.querySelector(`.toc a[href="#${e.target.id}"]`);
      if (lnk) lnk.classList.add('active');
    }
  });
}, { rootMargin: '-20% 0px -60% 0px' });
document.querySelectorAll('section[id]').forEach(s => observer.observe(s));
```

**Build:** `python build.py 03-gradient-descent` → assembla e apre nel browser.

### PDF → content.html workflow

1. Utente invia PDF della lezione
2. Agente legge il PDF, estrae struttura e contenuto
3. Genera `NN-slug/content.html` con front matter + sezioni
4. Utente esegue `python build.py NN-slug` per il file finale
5. Aggiorna `_nav.html` con il link alla nuova lezione

### content.html format — front matter + pure sections

```html
---
eyebrow: Lezione 03 · Apprendimento Automatico
title: Discesa del <em>Gradiente</em>
meta: A.A. 2025/26 · ~20 min lettura
tldr: Riassunto autonomo 2-4 righe. Supporta formule \(\theta^*\).
---

<section id="intro">
  <h2 id="intro">1. Introduzione</h2>   <!-- id su h2 → TOC auto-generato -->
  <p>...</p>
</section>

<section id="recap">
  <h2 id="recap">Flashcard di ripasso</h2>
  <div class="flashcards">...</div>
</section>
```

**Regola critica:** ogni `<h2>` deve avere `id="..."` → build.py genera il TOC automaticamente.

### MathJax macros (da preamble.tex)

Macro già configurate nel preamble HTML — usale come in LaTeX:

| Macro | Risultato |
|---|---|
| `\argmin` | operatorname argmin con limiti |
| `\argmax` | operatorname argmax con limiti |
| `\bm{v}` | vettore grassetto (da `bm` package) |
| `\dv{f}{x}` | derivata totale |
| `\pdv{f}{x}` | derivata parziale |

| `\norm{v}` | norma `‖v‖` |

### Algorithm block — analogo a algorithm2e

Colori nel design system: header `var(--g100)`, bordo `var(--oat)`, keyword `var(--clay-d)`.
Righe numerate automaticamente via CSS counter — nessun JS.

```html
<div class="algorithm">
  <div class="algorithm-header">
    <span class="algorithm-name">Algorithm 1</span>
    <span class="algorithm-caption">Gradient Descent</span>
  </div>
  <div class="algorithm-body">
    <div class="algo-line"><span><span class="algo-kw">Input:</span> \(X, y, \eta, T\)</span></div>
    <div class="algo-line"><span><span class="algo-kw">Output:</span> \(\theta^*\)</span></div>
    <div class="algo-line"><span><span class="algo-kw">for</span> \(t = 1, \ldots, T\) <span class="algo-kw">do</span></span></div>
    <div class="algo-line"><span><span class="algo-ind"></span>\(\theta \leftarrow \theta - \eta \nabla_\theta \mathcal{L}(\theta)\) <span class="algo-cm">// gradient step</span></span></div>
    <div class="algo-line"><span><span class="algo-kw">end for</span></span></div>
    <div class="algo-line"><span><span class="algo-kw">return</span> \(\theta\)</span></div>
  </div>
</div>
```

Token classes algorithm: `.algo-kw` (clay-d, grassetto), `.algo-fn` (slate), `.algo-cm` (g500 italic), `.algo-ind` (1.5em indent).

### Rules specific to university notes
- **Default layout: A4 fixed** — non responsive/fluid
- **Math: MathJax sempre** — mai plain text per equazioni
- **`<h2>` con id sempre** — obbligatorio per TOC automatico
- **Almeno un SVG per concetto visivo principale**
- **Flashcard in fondo** — sezione `id="recap"` fissa
- **Un file content.html per lezione** — mai accorpare argomenti

## Interaction Patterns (Level 3-4)

### Export button — mandatory at Level 3+
```js
document.getElementById('copyBtn').addEventListener('click', function() {
  const output = buildExport(); // costruisci il testo da esportare
  navigator.clipboard.writeText(output).then(() => {
    this.textContent = '✓ Copiato!';
    setTimeout(() => { this.textContent = 'Copia come markdown'; }, 2000);
  });
});
```

Without the export button, an interactive HTML is a toy. With it, the user's manual work feeds back into the next prompt.

## Single-File Discipline

- **One `.html` file. Always.** Inline `<style>` and `<script>`. No asset folder. No build step.
- Assets: inline SVG or `data:` URLs. Never `/img/foo.png`.
- External CDN allowed: MathJax only (for math rendering). No other external JS/CSS.
- The file must be openable by double-click — no server required.

## Rules (non-negotiable)

1. **State level explicitly** at top of response
2. **One file, always** — inline everything
3. **A4 fixed layout** for university notes (unless told otherwise)
4. **MathJax** for any math — never plain text equations
5. **Export button mandatory** at Level 3+
6. **SVG diagrams** preferred over descriptions for visual concepts
7. **Don't summarise away the substance** — preserve actual content
8. **Match level to task** — don't add sliders to a status report
